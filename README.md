# ระบบติดตามงานสื่อสารลูกค้า (Next.js)

CRM ภาษาไทยสำหรับทีมการตลาด/ครีเอทีฟ ใช้ติดตามงานสื่อสารที่ทีมทำให้ลูกค้าแต่ละราย ตามโครงสร้าง **ลูกค้า → ใบเสนอราคา (QT) → ชิ้นงาน**

แอปนี้เป็น frontend เท่านั้น ไม่มี backend และไม่มีระบบล็อกอิน ข้อมูลทั้งหมดเก็บใน `localStorage` ของเบราว์เซอร์ ภายใต้ key `teamTaskTracker.v2` (ดูหัวข้อ [ข้อจำกัด](#ข้อจำกัด))

โฟลเดอร์ `web/` คือเวอร์ชัน Next.js ของโปรเจกต์ ซึ่งพอร์ตมาจากแอปไฟล์เดียว `index.html` เดิมที่อยู่ใน root ของ repo (ดู README ที่ root สำหรับข้อมูลชุดนั้น)

## การรันโปรเจกต์

ต้องมี Node.js และ npm ติดตั้งไว้ก่อน จากนั้นรันคำสั่งต่อไปนี้จากโฟลเดอร์ `web/`:

```bash
npm install       # ติดตั้ง dependencies
npm run dev       # เริ่ม dev server ที่ http://localhost:3000
npm run build     # build สำหรับ production
npm run start     # รันเซิร์ฟเวอร์ production (ต้อง build ก่อน)
npm run lint      # ตรวจโค้ดด้วย ESLint
npm run test      # รัน test suite ด้วย Vitest
npm run typecheck # ตรวจชนิดข้อมูลด้วย TypeScript (tsc --noEmit)
```

คำสั่งเหล่านี้คัดลอกมาจาก `scripts` ใน `package.json` โดยตรง

### Stack ที่ใช้

อ้างอิงเวอร์ชันจริงใน `package.json`:

- **Next.js** `16.2.10` (App Router) พร้อม **React** `19.2.4`
- **Tailwind CSS** `^4` สำหรับสไตล์
- **zustand** `^5.0.14` สำหรับ state management และ persist ลง `localStorage`
- **chart.js** `^4.5.1` กับ **react-chartjs-2** `^5.3.1` สำหรับกราฟในหน้ารายงาน
- **lucide-react** สำหรับไอคอน
- **Vitest** `^4.1.9` กับ **@testing-library/react** สำหรับเทสต์
- **TypeScript** `^5`

## โมเดลข้อมูล

โครงสร้างข้อมูลหลักอยู่ใน `lib/types.ts` แบ่งเป็นสอง entity: **ลูกค้า** (`Customer`) และ **ชิ้นงาน** (`Item`) แต่ละชิ้นงานอ้างอิงลูกค้าเจ้าของผ่าน `customerId` และพกเลขที่ใบเสนอราคา (`qtNo`) กับเลขที่ใบวางบิล (`invNo`) ของตัวเอง ไม่มีตาราง QT แยกต่างหาก แอปจัดกลุ่มชิ้นงานตาม `qtNo` ให้เห็นเป็นชุดในหน้าชิ้นงานและรายงานแทน

### ลูกค้า (`Customer`)

- `name`, `province`, `salesOwner`: ชื่อ จังหวัด และเจ้าของงานขาย
- `contactPerson`, `phone`, `lineId`: ข้อมูลผู้ติดต่อ
- `color`: สีประจำลูกค้า (hex) ใช้แสดงจุดสีในรายการและบอร์ด

### ชิ้นงาน (`Item`)

ชิ้นงานมีสถานะอิสระ 3 มิติ ปรับได้แยกจากกัน:

- **การดำเนินการ** (`execStatus`): `not_started`, `in_progress`, `published`, `done`
- **ผลลัพธ์** (`resultStatus`): `not_collected`, `in_progress`, `achieved`
- **การจัดส่งรีพอร์ต** (`reportStatus`): `not_sent`, `sent`

หลังชิ้นงานหมดอายุ (`finishedDate` ผ่านมาแล้ว) ยังมีสถานะที่ 4 แยกต่างหากสำหรับติดตามการต่ออายุ:

- **การต่ออายุ** (`renewalStatus`): `pending` (รอต่ออายุ), `renewed` (ต่ออายุแล้ว), `lost` (ไม่ต่อ / เสียลูกค้า)

ฟิลด์อื่นที่สำคัญ:

- `channel`: ช่องทาง (`facebook`, `web`, `google`, `line`, `tiktok`, `youtube`, `other`)
- `price`: ราคา (รวม VAT 7% ตามชีตต้นฉบับของทีม) ใช้คำนวณรายได้ทุกที่ในแอป
- `target`, `actual`: เป้าหมายและผลลัพธ์แบบข้อความอิสระ
- `metricName`, `metricUnit`, `targetValue`, `actualValue`: ตัวชี้วัดเชิงตัวเลข (เช่น ยอดเข้าถึง, ยอดขาย) ใช้คำนวณ % บรรลุเป้าในหน้ารายงาน
- `rating`: คะแนนดาว 0-5
- `deadline`, `publishDate`, `finishedDate`, `reportSentDate`: วันที่ตามวงจรงาน
- `followUpDate`, `followUpNote`: การติดตามงานส่วนตัวของผู้รับผิดชอบ แยกจากวันที่ของงานเอง
- `priority`: ระดับความสำคัญ (`high`, `medium`, `low`)
- `progress`: ความคืบหน้า 0-100 ผูกกับ `execStatus` โดยอัตโนมัติเมื่อย้ายสถานะบนบอร์ด
- `checklist`, `activity`: รายการเช็กลิสต์ย่อยและประวัติการแก้ไขของชิ้นงาน

## มุมมองหลัก

### สิ่งที่ต้องทำ (Home)

หน้าแรกรวมงานค้างเป็น worklist 6 กลุ่ม พร้อมปุ่มดำเนินการด่วนในแต่ละแถว:

- **เลยกำหนดเผยแพร่**: งานที่ยังไม่เผยแพร่แต่ `publishDate` ผ่านมาแล้ว
- **ยังไม่เก็บผลลัพธ์**: งานที่เผยแพร่แล้วแต่ `resultStatus` ยังเป็น "ยังไม่เก็บผลลัพธ์"
- **ยังไม่ส่งรีพอร์ต**: งานที่เผยแพร่แล้วแต่ยังไม่ได้ส่งรีพอร์ต (มีปุ่มด่วน "ส่งแล้ว")
- **ใกล้หมดอายุ (30 วัน)**: งานที่ `renewalStatus` เป็น "รอต่ออายุ" และ `finishedDate` อยู่ในอีก 30 วันข้างหน้า
- **หมดอายุแล้ว — ต่ออายุ**: งานที่หมดอายุแล้วแต่ยังไม่ได้ตัดสินใจต่ออายุ (มีปุ่มด่วน "ต่ออายุแล้ว" / "ไม่ต่อ")
- **ติดตามวันนี้**: งานที่ตั้ง `followUpDate` ไว้ถึงกำหนดหรือเลยกำหนดแล้ว (มีปุ่มด่วน "ติดตามแล้ว")

ด้านบนของหน้ายังมีแถบ KPI แบบอ่านอย่างเดียว 5 ตัว: รายได้เดือนนี้ (พร้อม % เทียบเดือนก่อน), % บรรลุผล, % ส่งรีพอร์ต, คะแนนเฉลี่ย และจำนวนงานรอต่ออายุ (พร้อมอัตราต่ออายุ)

ทุก worklist เคารพตัวกรอง **งานของฉัน** (จำกัดเฉพาะลูกค้าของเจ้าของงานขายที่ล็อกอินอยู่) ที่ตั้งค่าได้จาก Sidebar

### ชิ้นงาน (Items)

หน้านี้แสดงชิ้นงานแบบรายการหรือบอร์ด สลับได้จากปุ่ม **รายการ / บอร์ด**:

- ตัวกรอง: ค้นหา, ลูกค้า, ช่องทาง, หมวดสถานะ (การดำเนินการ/ผลลัพธ์/รีพอร์ต) พร้อมค่าสถานะในหมวดนั้น, เจ้าของงานขาย, จังหวัด, ช่วงวันที่ (พร้อมปุ่มลัด เดือนนี้ / 30 วันล่าสุด / ไตรมาสนี้ / ปีนี้) และตัวกรอง **งานค้าง** (เฉพาะงานที่เกินกำหนด)
- แต่ละแถวชิ้นงานแสดงป้าย **เกินกำหนด** หรือ **ใกล้ครบกำหนด** (ภายใน 3 วัน) ตามความจำเป็น รวมถึงเตือนเมื่อลิงก์งานชี้ไปจังหวัดอื่นที่ไม่ตรงกับลูกค้า
- มุมมองบอร์ด (Kanban) สลับมิติที่ใช้จัดคอลัมน์ได้ (การดำเนินการ/ผลลัพธ์/รีพอร์ต) ลากวางการ์ดเพื่อเปลี่ยนสถานะ
- มุมมองรายการรองรับการเลือกหลายรายการพร้อมกัน (bulk select) แล้วทำเครื่องหมายเผยแพร่แล้ว, บรรลุผล หรือส่งรีพอร์ตแล้วในครั้งเดียว

### Calendar

ปฏิทินรายเดือนหรือมุมมองตาราง (agenda) สลับฐานวันที่ที่ใช้วางชิ้นงานได้ (Publish Date / Deadline / Finished Date) เน้นงานเกินกำหนดด้วยสีแดง และแยกโซนแสดงงานที่ไม่มีวันที่ในฟิลด์ที่เลือก

### ลูกค้า (Customers)

รายการลูกค้าแบบการ์ด แสดงข้อมูลผู้ติดต่อ (ผู้ติดต่อ, เบอร์โทร, LINE ID), รายได้รวม, % บรรลุผล, ความคืบหน้าเฉลี่ย และจำนวนงานค้าง/โอกาสต่ออายุต่อราย ค้นหาและกรองตามเจ้าของงานขาย/จังหวัดได้ พร้อมตัวเลือกเรียงลำดับหลายแบบ แต่ละการ์ดเปิดหน้าต่างแก้ไขลูกค้า (แก้ข้อมูลติดต่อ, สีประจำลูกค้า และ **รวมกับลูกค้าอื่น** สำหรับกรณีข้อมูลซ้ำ ซึ่งย้ายชิ้นงานทั้งหมดไปยังลูกค้าที่เลือกแล้วลบรายการเดิมทิ้ง ทำย้อนกลับไม่ได้) หรือเปิดชิ้นงาน/รีพอร์ตของลูกค้ารายนั้นโดยตรง

### Command Palette

กด `Ctrl/⌘K` (หรือ `/` เมื่อไม่ได้พิมพ์อยู่ในช่องข้อความ) เพื่อเปิดหน้าต่างค้นหาคำสั่ง ใช้สลับหน้า, เพิ่มชิ้นงานใหม่, โหลดข้อมูลจริงของทีม หรือ Export CSV/JSON ได้โดยไม่ต้องใช้เมาส์

### รายงาน (Report / Analytics)

หน้ารายงานมี 2 แท็บ คือ **ภาพรวม** และ **ยอดขาย** กรองตามเดือน publish ได้ และพิมพ์เป็น PDF ผ่านปุ่ม **พิมพ์ / บันทึก PDF** (`window.print`)

แท็บภาพรวมแสดง KPI 5 ตัวพร้อมส่วนต่างเทียบเดือนก่อน (month-over-month) แล้วตามด้วยส่วนที่ยุบ/ขยายได้ (collapsible) 4 กลุ่ม:

- **การเงิน & รายได้**: พยากรณ์รายได้ล่วงหน้า (คาดการณ์จากงานที่เสนอราคาแล้วบวกโอกาสต่ออายุ), ตรวจสอบข้อมูลราคาที่ผิดปกติ (ราคาติดลบ หรืองานที่ส่งมอบแล้วแต่ทั้งใบเสนอราคารวมเป็น 0 บาท) และรายการใบเสนอราคาที่ยังวางบิลไม่ครบ (QO → INV)
- **การต่ออายุ & สุขภาพลูกค้า**: อัตราการต่ออายุและ pipeline งานใกล้/เลยกำหนดต่ออายุ, คะแนนสุขภาพบัญชีลูกค้ารายตัว (เสี่ยง/เฝ้าระวัง/แข็งแรง พร้อมเหตุผล) และความเสี่ยงกระจุกตัวของรายได้ (ต่อเจ้าของงานขายรายเดียว และต่อลูกค้ารายใหญ่)
- **ประสิทธิภาพงาน**: funnel การผลิตงาน (เผยแพร่ → เก็บผลลัพธ์ → บรรลุผล → ส่งรีพอร์ต), ผลงานแยกตามช่องทาง/เจ้าของงานขาย/ประเภทงาน, ผลเทียบเป้าหมายเชิงตัวเลข และ SLA การส่งรีพอร์ต (ระยะเวลาเฉลี่ย/มัธยฐานจากวันจบงานถึงวันส่งรีพอร์ต เทียบเป้า 7 วัน)
- **กราฟ**: กราฟ Chart.js 7 แบบ ครอบคลุมสถานะการดำเนินการ/ผลลัพธ์ รายได้แยกช่องทาง/เจ้าของงานขาย/เดือน Top 10 ลูกค้าตามรายได้ และจำนวนชิ้นงานแยกช่องทาง

นอกจากรายงานรวม แต่ละลูกค้ายังมีรีพอร์ตเฉพาะราย (เปิดจากปุ่ม **ดูรีพอร์ต** ในหน้าลูกค้า) จัดกลุ่มชิ้นงานตาม QT หรือ INV และพิมพ์เป็น PDF ได้เช่นกัน

## การนำเข้า/ส่งออก

ปุ่มเครื่องมือข้อมูลอยู่ใน Sidebar:

- **Import CSV/JSON**: รับไฟล์ `.csv` ตามรูปแบบชีตติดตามงานประจำเดือนของทีม (หัวตารางค้นหาจากแถวที่มีคอลัมน์ "ชื่อลูกค้า" รองรับราคาที่คั่นหลักพันด้วยจุลภาคและวันที่แบบ `d-Mon-yyyy`) หรือไฟล์ `.json` ที่ export จากแอปนี้เอง ระบบจะแสดงจำนวนลูกค้า/ชิ้นงานที่จะนำเข้าและขอยืนยันก่อนเขียนทับข้อมูลปัจจุบันเสมอ
- **Export CSV**: ได้ไฟล์ `customer-communication-tracking.csv` ตามลำดับคอลัมน์เดียวกับชีตต้นฉบับของทีม ไฟล์มี UTF-8 BOM นำหน้า ทำให้ Excel เปิดข้อความภาษาไทยได้ถูกต้องโดยไม่ต้องแปลง encoding เอง
- **Export JSON**: ได้ไฟล์ `customer-communication-backup.json` เก็บข้อมูลครบทุกฟิลด์ ใช้สำรอง/กู้คืนข้อมูลแบบเต็ม

**ข้อควรระวัง**: การ export เป็น CSV คงคอลัมน์ตามชีตต้นฉบับของทีมเท่านั้น จึงไม่รวมฟิลด์ที่เพิ่มขึ้นมาในแอปนี้ เช่น `notes`, ข้อมูลผู้ติดต่อของลูกค้า (`contactPerson`/`phone`/`lineId`), `renewalStatus`, ตัวชี้วัดเชิงตัวเลข (`metricName`/`metricUnit`/`targetValue`/`actualValue`) และการติดตามงาน (`followUpDate`/`followUpNote`) ฟิลด์เหล่านี้อยู่ใน Export JSON เท่านั้น หากต้องการสำรองข้อมูลครบถ้วนให้ใช้ **Export JSON**

## สถาปัตยกรรมโดยย่อ

- **`lib/derived.ts`**: ฟังก์ชันคำนวณล้วน (pure function) ที่แปลงรายการลูกค้า/ชิ้นงานเป็นข้อมูลที่หน้าจอต่าง ๆ ใช้แสดงผล เช่นตัวกรอง สถิติ Dashboard และทุกตัวชี้วัดในหน้ารายงาน ไม่มีการเรียก state หรือ side effect ใด ๆ ในไฟล์นี้
- **`lib/store.ts`**: state store ด้วย zustand พร้อม middleware `persist` ใช้ตัวจัดเก็บ (storage adapter) ที่เขียนเข้า `localStorage` เอง แทนตัวช่วยมาตรฐานของ zustand เพื่อดักจับข้อผิดพลาดตอนเขียน (เช่น พื้นที่จัดเก็บเต็ม) และ normalize ข้อมูลทุกครั้งที่โหลด
- **`lib/normalize.ts`**: ชุดฟังก์ชัน self-healing ที่รับประกันว่าข้อมูลลูกค้า/ชิ้นงานที่โหลดจาก `localStorage` หรือไฟล์นำเข้า จะมีโครงสร้างและชนิดข้อมูลที่ถูกต้องเสมอ แม้ข้อมูลเดิมจะไม่สมบูรณ์
- **Error boundary**: `app/error.tsx` ดักข้อผิดพลาดในตัวแอป และ `app/global-error.tsx` ดักข้อผิดพลาดร้ายแรงในระดับ root layout ทั้งสองแสดงข้อความว่าอยากลองใหม่ได้ทันทีและข้อมูลยังอยู่ในเบราว์เซอร์
- **แจ้งเตือนพื้นที่จัดเก็บเต็ม**: หากเขียน `localStorage` ไม่สำเร็จ (`components/StorageAlert.tsx`) แอปจะแสดงข้อความเตือนค้างไว้จนกว่าผู้ใช้จะกด Export JSON สำรองข้อมูล
- **Focus trap**: `lib/useFocusTrap.ts` ใช้ในหน้าต่าง modal และ Command Palette เพื่อให้ปุ่ม Tab วนอยู่ภายใน dialog ที่เปิดอยู่ (a11y)
- **Test suite**: ทดสอบด้วย Vitest ที่ `lib/__tests__/` ปัจจุบันมี 109 เทสต์ ผ่านทั้งหมด (25 ไฟล์เทสต์ รันด้วย `npm run test`)

## ข้อจำกัด

- ข้อมูลอยู่เฉพาะเบราว์เซอร์และโปรไฟล์ที่ใช้งาน ไม่มี cloud sync และไม่มีระบบบัญชีผู้ใช้
- ล้างข้อมูลเว็บไซต์ เปลี่ยนเบราว์เซอร์ หรือเปลี่ยนเครื่องแล้ว ข้อมูลจะไม่ตามไปด้วย
- หากพื้นที่ `localStorage` เต็มหรือเบราว์เซอร์ปิดกั้นการเขียน (เช่นโหมดส่วนตัว) แอปจะเตือนทันทีให้ Export JSON สำรองข้อมูลก่อนรีเฟรชหน้า
